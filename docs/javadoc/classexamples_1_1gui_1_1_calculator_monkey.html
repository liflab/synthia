examples.gui.CalculatorMonkey Class Reference

Detailed Description

Performs monkey testing over a simple calculator, using Synthia's Monkey.

The monkey is instantiated with an equal probability of clicking on any of the calculator's 16 buttons at any moment. It runs until the calculator throws an exception, after which it will attempt to shrink the sequence –i.e., to find a subsequence that still causes the exception to be thrown.

The picker for actions requires a Picker<Float> in order to generate a subsequence. Instead of supplying it with a uniform RandomFloat in the interval [0,1], we rather pass a BiasedRandomFloat that gives preference to values closer to 1. This has for effect that sub-sequences will contain more actions from the end of the reference sequence instead of the beginning. This is based on the intuition that errors are more likely to occur because of actions close to the occurrence of the exception.

Depending on the seed and on the faults enabled in the Calculator, four outcomes are likely:

  1. no error found
  2. number format exception
  3. overflow
  4. division by zero
